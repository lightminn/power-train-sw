# SW 팀 진행 보고 — 3월 ~ 8월 말 (대회 일정)

> **작성일:** 2026-05-30
> **작성:** 김광민 (SW · Design/control)
> **대상:** 회의 공유용 (진행 상황 + 추후 계획 거시 요약)

> **Historical snapshot:** 아래 상태표와 일정은 2026-05-30 당시 기록이다. 현재 정본은
> `docs/plans/2026-07-02-autonomous-driving-kickoff.md`를 따른다. 2026-07-10 기준 단일 CAN
> 10모터·4WS 텔레옵·WP4 ROS 왕복·WP5 chassis_node 실기 HIL까지 완료했으며, 센서는
> L515=파워트레인, D435i=로봇팔 전용으로 분리됐다. v4 최적화 질량은 50 kg로 확정했고
> 86 kg 재최적화는 하지 않는다.

---

## 0. 한 줄 요약

SW팀은 3월 이후 (1) 다리 치수 최적화를 v4까지 진행했고, (2) 모터·센서·비전 부품을
**개별 단위로** 잡고 게임패드까지 굴리는 단계를 마쳤습니다. 6~8월에는 이것들을
한 줄로 이어 붙여 **실제 6륜 로봇 한 대가 굴러가는** 단계로 진행합니다.

---

## 1. 우리가 만드는 것 (1분 요약)

ZETIN 방어로봇은 **6륜 로커보기 서스펜션 로봇**. SW는 두 트랙으로 갑니다.

| 트랙 | 풀어 쓰면 |
|---|---|
| **(A) 다리 치수 최적화** | 로커-보기 다리 길이·각도 같은 치수를 컴퓨터로 7가지 지형(계단/나무/거친지면/단턱/곡선램프/15°·30° 경사) 시뮬 돌려서 가장 잘 넘는 값으로 정함 |
| **(B) 로봇 제어 SW** | 모터·조향·센서·카메라·게임패드를 묶어 로봇이 굴러가게 하는 프로그램 |

용어:
- **ODrive** = 구동 모터를 돌리는 제어보드.
- **AK40-10 / AK45-36** = 조향 모터. 테스트는 AK40, 실전은 AK45.
- **BL70200** = 실전 구동 모터(인휠형).
- **Jetson Orin Nano** = 로봇 두뇌 보드.
- **CAN / USB** = 두뇌와 모터를 잇는 통신선 두 종류.
- **YOLO** = 카메라 영상에서 물체를 자동으로 찾는 AI.

## 2. 3월 ~ 5월에 한 일 (큰 흐름)

### (A) 다리 치수 최적화 — 서버에서

MATLAB 원본 분석 → Python(CPU) 포팅 → GPU 포팅(v3, 14차원·4지형) → **v4**(15차원·7지형, 물리 모델 보정)까지 진행.
- 결과 파라미터·시뮬 애니메이션 4지형분 산출.

### (B) 로봇 제어 SW — 개별 부품 단위

- **모터 단품 구동** — ODrive 캘리브레이션, 조향(AK) CAN 제어, 게임패드 원격조종.
- **비전 + 모터 추종 데모** — Jetson 카메라로 사람 잡으면 모터가 따라 돔.
- **영상 스트리밍** — Jetson → 노트북 H.264 송신·수신.
- **웹 진단 GUI** — 브라우저로 모터를 켜고/끄고/튜닝, 모든 실하드웨어 시험에 사용.
- **코너 모듈 (조향+구동 협조 제어)** — 로봇 1/6 빌딩블록 라이브러리, 실하드웨어 검증 완료. 게임패드 텔레옵 데모까지 동작.
- **US-100 충돌방지 모듈** — 거리 → "안전/주의/멈춤" 판정 모듈. 오늘 중 완성 예정.

### 문서·표준화

AK45-36(실전) 사양, 로봇 총중량 86kg(여유 100kg), BL70200 단일 CAN 버스 통합 방침을 README·CLAUDE·Notion에 모두 반영해 정합화.

### Phase 1 — 기반 (하드웨어 + 골격)

- **T1. 조향 USB→CAN 통합** — 구동(USB)+조향(CAN) 혼합을 단일 CAN 버스로 일원화.
- **T2. 실전 모터 셋업** — 실전 모터(BL70200·AK45-36) 입고분 캘리브레이션·튜닝.
- **T3. FSM (메인 상태머신)** — 로봇 전체 동작 골격을 먼저 잡음. `IDLE → MANUAL → AUTO → SAFE_STOP → FAULT` 전이, 안전 신호·사용자 입력에 따른 모드 스위치. 이후 모든 기능이 이 골격에 꽂힘.

### Phase 2 — 6륜 통합

- **T4. 6륜 다중 코너** — 코너 모듈 ×4(조향)/×6(구동) 동시 운용. 한 CAN 버스에서 충돌 없이.
- **T5. 4WS 애커만 조향** — 차체 속도·조향 명령을 각 코너 명령으로 분배하는 키네마틱스 레이어.
- **T6. 원격조종 완성** — 게임패드 + 실시간 영상 + 진단 정보를 하나의 GUI로 통합. 6륜 전체 본격 텔레옵.

### Phase 3 — 상위 기능 + 통합

- **T7. 자율주행 기능** — 미션 시퀀스(웨이포인트 추종 + 장애물 회피). 정해진 코스 자동 주행 수준으로 한정(전체 SLAM 자율은 범위 밖).
- **T8. 로봇팔 통합** — 로봇팔 팀 산출물과 인터페이스·통신·전원·안전 협조 정의. 차체↔팔 명령 채널 + 동시 동작 시 안전 룰.
- **T9. 비전+안전 융합** — YOLO 검출 + US-100 충돌방지를 하나의 안전·인지 레이어로 묶어 FSM에 입력.
- **T10. 메인 컨트롤러 통합** — 부팅·캘리·안전·텔레옵·자율·팔 협조를 FSM 위에 모두 연결. 로그/블랙박스 포함.
- **T11. 필드 테스트/안정화** — 7가지 지형 실주행 + 현장 튜닝.

## 5. SW팀이 필요로 하는 재료

### 하드웨어 (실전용, 전장팀 의존)

| 품목 | 수량 | 용도 |
|---|---|---|
| BL70200 구동 모터 | 6 | 실전 6륜 구동 |
| AK45-36 조향 모터 | 4 | 실전 4WS 조향 |
| ODrive 3.6 | 6 | 구동 모터 드라이버 (현재 1대 보유) |
| US-100 초음파 센서 | 1~3 | 보조 충돌방지·추락방지 |
| **로봇팔 통신 채널** | — | 로봇팔 팀과 협의 (TCP·CAN 또는 ROS 토픽 등) |

### 케이블·통신·전원

| 품목 | 용도 |
|---|---|
| CAN 트랜시버·케이블·종단저항 | 단일 CAN 버스 데이지체인 |
| ODrive↔AK 커스텀 CAN 케이블 | 커넥터 직결 불가 — 제작 필요 |
| 비상정지 스위치 (estop) | 안전 차단 |
| BMS / 파워 모듈 | 48V 배터리 분배 |
