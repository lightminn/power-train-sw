# A2c 배치 구현 계획 — 무SSH 브링업 (oneshot 유닛·비콘·NVM 캘리 영속화·부팅 자격화)

> **For agentic workers:** superpowers:subagent-driven-development 또는 executing-plans.
> **레포 관례:** Codex 위임(git 금지 — 커밋은 리뷰어) + 3환경 + 젯슨 실기.

**Goal:** 스펙 r6 §3.4 — 대회 당일 SSH 없이 로봇을 세우는 브링업 자동화. 부팅
자동기동(기존 `restart: unless-stopped` 재사용) + oneshot 유닛(can0 pinmux +
env/토큰 preflight) + crash-loop 비콘(기존 pdist 센더 확장) + 캘리 NVM 영속화 +
**chassis 소유 부팅 자격화 게이트(모든 arm 경로 통과)**.

**Architecture:** 실모터가 필요 없는 SW 구조·계약을 지금 구현·단위테스트하고,
실제 NVM 쓰기·캘리 실행·조립 전 게이트는 벤치 이월로 분리한다. 순수 코어 우선:
①`preflight.py`(env/토큰/stop_mm provenance 검증 — 순수) ②`boot_qualification.py`
(축별 플래그+보드 지문 판정 — 순수, ChassisManager에 주입) ③`board_registry`
(시리얼→노드쌍 매핑) 위에 얇은 실행부. 비콘은 pdist 센더에 유닛 상태 필드 추가.

**Spec:** r6 §3.4. 기준선(A2b 후): 호스트 254 / dev 1026+2skip / ros 458 / 젯슨 458.

## Global Constraints

- A2a/A2b Global Constraints 승계. 토큰·비번 실값 커밋 금지.
- **실모터 회전 수반 작업은 이 배치에서 실행하지 않는다** — 코드·계약·단위테스트만.
  실제 NVM 영속화·캘리 lifecycle 실행·3회 전원사이클 게이트는 **벤치 이월**(FULL
  HIL 규율, 사용자 물리 확인). fw 0.5.1 전제(HALL polarity 상태는 0.5.2+ — 근거 금지).
- systemd/compose는 젯슨 배포 파일 — 문법·구조만 바꾸고 젯슨 실증은 배치 말.
- 젯슨 실기 검증(비회전): oneshot preflight 통과/실패 동작, 비콘 유닛-상태 필드
  콘솔 수신, 부팅 자격화 게이트가 미자격 시 arm 거부(FAKE 모터로).

@dataclass(frozen=True)
class PreflightResult:
    ok: bool
    failures: tuple          # 사람이 읽는 실패 사유
    warnings: tuple          # 예: BENCH provenance
    stop_mm: float | None
    provenance: str | None   # "BENCH" | "COMMISSIONED"
def check_preflight(env: dict, token_dir_lister) -> PreflightResult
    # env: {STOP_MM, STOP_MM_PROVENANCE, ...} (문자열), token_dir_lister:
    #   ()->set[str] (존재하는 토큰 파일명). 검사:

    #   - STOP_MM_PROVENANCE ∈ {BENCH, COMMISSIONED} 아니면 failure;
    #     BENCH면 warning(콘솔 배지용)
    #   - ops_console.token·ops_controller.token 최소 1개 존재 아니면 failure
STOP_MM_MIN = 50.0
STOP_MM_MAX = 2000.0

### Task 3: ChassisManager 자격화 게이트 — 모든 arm 경로 통과

**Interfaces:**
- `ChassisManager(__init__, ..., qualification_gate=None)` — `qualification_gate`
  는 `()->QualificationResult|None` 콜러블(None=미설정, 게이트 없음=현행 동작
  불변). `arm()`은 mode==IDLE·¬estop 검사 **이후**, 게이트가 설정돼 있고
  `gate().qualified is False`면 `estop("boot_qualification", 사유)` 후 False 반환
  (arm 우회 불가 — broker·`~/arm`·chord 전부 이 arm()을 지난다). **power-session
  무효화(Codex r2-9)**: 게이트 결과가 stale(마지막 판정 후 ODrive heartbeat
  리셋 감지)이면 재판정 전 미자격 취급 — 게이트 콜러블 내부 책임(노드가 구현).
- chassis_node: `boot_qualification_enabled` 파라미터(기본 False). True면 실기
  경로에서 축 리포트 수집(ODrive heartbeat/플래그) → `qualify_axes` → 게이트 주입.
  FAKE 모터는 항상 qualified(벤치 개발 무영향).
