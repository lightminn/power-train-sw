## 1. 한 줄 요약 — 무엇을 만드나요?

로봇 앞에 달린 **거리 센서(US-100)** 하나로 앞에 장애물이 얼마나 가까운지 계속 재서,
**"안전 / 주의 / 멈춤"** 셋 중 하나로 알려주는 작은 프로그램입니다.

중요한 점: 이 프로그램은 **모터를 직접 멈추지 않습니다.** 그냥 "지금 위험해요"라고
**알려주기만** 합니다. 실제로 멈출지는 로봇을 운전하는 다른 프로그램이 그 신호를
보고 결정합니다. (이렇게 역할을 나누면, 다른 사람이 만드는 운전 프로그램과 따로따로
개발·시험할 수 있어서 편합니다.)

비유하면, **자동차 후방 경고음**과 비슷합니다. 경고음 장치는 "삐— 삐— 삐"(가까움)만
울리고, 실제로 브레이크를 밟는 건 운전자죠. 여기서 이 모듈은 그 "경고음 장치"입니다.

---

## 2. US-100 센서가 뭔가요?

손가락만 한 **초음파 거리 센서**입니다. 박쥐처럼 소리(초음파)를 쏘고, 앞 물체에
부딪혀 돌아오는 시간을 재서 거리를 알려줍니다.

- 로봇 컴퓨터(Jetson)와 **선(UART)** 으로 연결됩니다. 연결 통로 이름은 `/dev/ttyTHS1`,
  통신 속도 설정값은 `9600` 입니다. (이건 그냥 정해진 값이라 그대로 쓰면 됩니다.)
- 사용법: 컴퓨터가 센서에게 "재 줘"라는 신호(숫자 `0x55`)를 보내면, 센서가 거리를
  **밀리미터(mm)** 로 답해 줍니다. (1m = 1000mm)
- 잴 수 있는 범위는 대략 **2cm ~ 4m** 입니다. 이 범위를 벗어나거나 답이 없으면
  "못 쟀다"로 처리합니다.

> **이미 만들어 둔 참고 코드:** `motor_control/sensors/us100_robust.py` 에 이 센서를
> 안정적으로 읽는 방법이 들어 있습니다. (Jetson에서 첫 글자가 깨지는 버그가 있어서,
> 신호 앞에 `0xFF` 라는 더미 값을 몇 개 붙여 보내는 요령이 들어 있어요. 우리도 이
> 방법을 그대로 가져다 씁니다. 새로 발명할 필요 없습니다.)

## 3. 핵심 개념 3가지 (꼭 이해하고 가기)

### (1) 판정 3단계 — "안전 / 주의 / 멈춤"

거리에 따라 셋 중 하나를 정합니다. 영어로는 각각 `safe`, `warn`, `stop` 이라고 부릅니다.

| 단계 | 영어 이름 | 언제 | 예시(기본값) |
|------|-----------|------|--------------|
| 안전 | `safe` | 충분히 멀다 | 40cm 보다 멀 때 |
| 주의 | `warn` | 슬슬 가깝다 | 20~40cm |
| 멈춤 | `stop` | 너무 가깝다 | 20cm 이내 |

이 거리 기준(40cm, 20cm)은 **나중에 바꿀 수 있는 설정값**입니다. 처음엔 이 값으로
시작하고, 실제로 돌려보며 적당히 조절하면 됩니다.

### (2) "모르면 일단 멈춤" — 안전을 위한 기본 태도 (영어로 fail-safe)

센서가 고장 났거나, 선이 빠졌거나, 답이 안 오면 어떻게 할까요?
→ **"잘 모르겠으면 위험하다고 보고 `멈춤`으로 판정"** 합니다.

이유: 안전 장치는 "확실히 안전할 때만 안전"이라고 해야 합니다. 센서가 먹통인데
"안전"이라고 하면 로봇이 벽으로 돌진할 수 있으니까요.

다만 US-100은 가끔 한두 번 답을 놓칠 때가 있어서, **연속으로 몇 번(기본 3번)** 실패하면
그때 `멈춤`으로 바꿉니다. (한 번 놓쳤다고 바로 멈추면 너무 예민해서요. 직전 판정을
잠깐 유지하다가, 계속 실패하면 멈춤.)

### (3) 깜빡임 방지 — "여유 두기" (영어로 hysteresis, 히스테리시스)

거리가 기준선(예: 20cm) 바로 근처에서 미세하게 흔들리면, 판정이
`멈춤→주의→멈춤→주의`로 **깜빡깜빡** 바뀔 수 있습니다. 보기 안 좋고 동작도 불안정하죠.

그래서 **풀어줄 때(덜 위험한 쪽으로 갈 때)만 약간의 여유**를 둡니다.
- 예: `멈춤`에서 벗어나려면 20cm가 아니라 **20cm + 3cm(여유) = 23cm** 보다 멀어져야 함.
- 반대로 **위험해지는 쪽으로는 즉시** 바꿉니다 (안전이 우선이니까 머뭇거리지 않음).

이 "여유" 기본값은 3cm(30mm)입니다.

## 4. 프로그램 구조 — 어떤 파일들로 나누나요?

여러 일을 한 파일에 다 넣지 않고, **하는 일별로 파일을 나눕니다.** 그래야 각 파일이
짧고 이해하기 쉽고, 한 군데를 고쳐도 다른 데가 안 깨집니다.

새 폴더: `motor_control/safety_us100/`

```
safety_us100/
├── config.py          # 설정값 모음 (거리 기준, 여유, 연결 포트 등)
├── verdict.py         # "판정 결과" 모양 정의 (단계 + 거리)
├── evaluator.py       # 거리 → 판정 계산 (순수 계산만, 센서 없이 동작)
├── us100.py           # 실제 센서에서 거리를 읽어오는 부분 (하드웨어 담당)
├── fake_sensor.py     # 가짜 센서 (하드웨어 없이 시험할 때 사용)
├── safety_monitor.py  # 위 조각들을 합쳐 "계속 감시"하는 본체
├── demo.py            # 단독 실행해서 눈으로 확인하는 프로그램
├── README.md          # 사용 설명서
└── tests/test_safety.py   # 자동 시험 코드
```

각 파일을 쉬운 말로 설명하면:

- **config.py** — "거리 기준은 몇 cm? 여유는 몇 cm? 센서는 어느 선에 연결?" 같은
  **숫자·설정을 한곳에** 모아 둔 파일. 나중에 값만 여기서 바꾸면 됨.
- **verdict.py** — 판정 결과의 **모양(틀)** 을 정해 둔 파일. 결과는 딱 두 가지 정보:
  `단계`(안전/주의/멈춤)와 `거리`(mm, 못 쟀으면 비어 있음).
- **evaluator.py** — "거리 숫자를 받으면 어느 단계인지 계산"하는 부분. **센서가 없어도**
  돌아가는 순수 계산이라, 시험하기 가장 쉬움.
- **us100.py** — 진짜 센서에 말 걸어 거리를 받아오는 부분. (3절의 참고 코드 방식 사용.)
- **fake_sensor.py** — 진짜 센서 대신 "내가 미리 정해둔 거리들을 차례로 내놓는 가짜
  센서". 하드웨어 없이도 프로그램이 잘 도는지 시험할 때 씀.
- **safety_monitor.py** — **본체.** 센서에서 거리 읽기 → evaluator로 판정 계산 →
  결과 보관, 이 과정을 반복. "모르면 멈춤" 처리도 여기서 함.
- **demo.py** — 실제로 켜서 화면에 "지금 30cm, 주의" 같은 걸 계속 찍어주는 프로그램.
  팀원이 거리 기준을 조절하며 눈으로 확인하는 용도.

> **중요(역할 분리):** 이 폴더의 코드는 **모터 제어 코드(`corner_module`)나 화면 도구
> (`motor_gui`)를 건드리지 않습니다.** 완전히 따로 동작합니다. 그래서 다른 사람 작업과
> 충돌 없이 혼자 개발·시험할 수 있습니다.

## 5. 판정 결과의 모양 (verdict.py)

판정 결과는 딱 두 가지만 담습니다.

```python
# 단계 이름 (오타 방지용으로 미리 정해둠)
SAFE, WARN, STOP = "safe", "warn", "stop"

@dataclass(frozen=True)   # "값 묶음 한 개"를 만드는 파이썬 기능 (수정 불가 묶음)
class Verdict:
    level: str                  # 단계: "safe" / "warn" / "stop"
    distance_mm: float | None   # 잰 거리(mm). 못 쟀으면 None(=비어 있음)
```

- `level` 이 무슨 단계인지 알려줍니다.
- `distance_mm` 이 실제 거리입니다. 센서를 못 읽었을 땐 `None`(비어 있음)이라,
  소비측은 "거리가 None이고 단계가 stop이면 = 센서 문제로 멈춤"이라고 구분할 수 있습니다.

> `@dataclass`, `None` 같은 단어가 낯설면 지금은 "결과는 단계 + 거리, 두 개를 담은
> 상자"라고만 이해해도 충분합니다.

## 6. 거리 → 판정 계산 규칙 (evaluator.py)

함수 하나가 **거리 숫자**(와 직전 단계)를 받아 **단계**를 돌려줍니다.

규칙 (위에서부터 순서대로 확인):
1. 거리가 **비어 있음(None)** 이면 → **멈춤(stop)**  ("모르면 위험")
2. 거리가 **멈춤 기준(기본 200mm) 이하**면 → **멈춤(stop)**
3. 거리가 **주의 기준(기본 400mm) 이하**면 → **주의(warn)**
4. 그 외(충분히 멀다)면 → **안전(safe)**

여기에 **깜빡임 방지(여유)** 를 더합니다:
- 더 **위험한** 단계로 갈 때 → **즉시** 바꿈.
- 더 **안전한** 단계로 풀 때 → 기준보다 **여유(기본 30mm)만큼 더** 멀어져야 바꿈.

(예: 지금 `멈춤`인데 거리가 210mm로 살짝 멀어져도 아직 `멈춤` 유지. 230mm를 넘어야
`주의`로 풀림.)

이 파일은 **센서가 없어도 되는 순수 계산**이라, 가짜 숫자만 넣어서 자동 시험하기 쉽습니다.

## 7. 감시 본체 (safety_monitor.py)

이게 실제로 "계속 돌면서 감시"하는 부분입니다. 두 가지 동작을 합니다.

- **`tick()`** — "한 번 확인해" 라는 뜻. 호출하면:
  1. 센서에서 거리를 한 번 읽는다.
  2. 잘 읽혔으면 → evaluator로 단계를 계산하고, 실패 횟수를 0으로 초기화.
  3. 못 읽었으면 → 실패 횟수를 1 늘린다.
     - 아직 **연속 실패가 기준(기본 3번) 미만**이면 → 직전 판정을 그대로 둠.
     - **기준에 도달**하면 → 판정을 `멈춤(거리 비어 있음)`으로 바꿈. (fail-safe)
- **`verdict()`** — "지금 판정이 뭐야?" 라고 물으면 최신 판정(`Verdict`)을 돌려줌.

사용하는 쪽(운전 프로그램)은 자기 반복문 안에서 `tick()` 한 번 부르고 `verdict()`로
결과를 확인하면 됩니다. (별도의 복잡한 백그라운드 동작 없음 — 이해하기 쉬움.)

## 8. 설정값 (config.py)

바꿀 수 있는 숫자들을 한곳에 모읍니다. 처음엔 아래 기본값으로 시작하세요.

| 설정 이름 | 기본값 | 뜻 |
|-----------|--------|-----|
| `warn_mm` | 400 | 이 거리(mm) 이하면 "주의" (=40cm) |
| `stop_mm` | 200 | 이 거리(mm) 이하면 "멈춤" (=20cm) |
| `hysteresis_mm` | 30 | 깜빡임 방지 여유 (=3cm) |
| `fail_stop_count` | 3 | 연속 몇 번 실패하면 멈춤으로 볼지 |
| `port` | `/dev/ttyTHS1` | 센서가 연결된 선 이름 (그대로 두기) |
| `baud` | 9600 | 통신 속도 (그대로 두기) |

> 규칙: `stop_mm` < `warn_mm` 이어야 합니다 (멈춤 기준이 주의 기준보다 가까워야 말이 됨).

## 9. 시험 방법 (tests/test_safety.py + fake_sensor.py)

하드웨어(진짜 센서)가 없어도 프로그램이 맞게 동작하는지 **자동으로** 확인합니다.
이를 위해 **가짜 센서(`FakeUs100`)** 를 씁니다 — "내가 미리 정한 거리들을 차례로
내놓는" 가짜 부품입니다.

확인할 항목:
1. 멀면 `안전`, 중간이면 `주의`, 가까우면 `멈춤` 이 나온다.
2. 거리가 비어 있으면(None) `멈춤` 이 나온다. (모르면 멈춤)
3. 한두 번 실패는 직전 판정 유지, **연속 3번** 실패하면 `멈춤`.
4. 기준선 근처에서 거리가 흔들려도 판정이 깜빡거리지 않는다. (여유 동작 확인)
5. 멈춤이었다가 거리가 충분히 멀어지면 다시 `안전`으로 돌아온다.
6. 판정 결과가 `단계 + 거리` 모양으로 잘 나온다.

> 진짜 센서를 읽는 `us100.py` 는 자동 시험이 어렵습니다(실제 부품 필요). 이건 로봇에
> 연결해 `demo.py` 로 **직접 눈으로** 확인합니다.

## 10. 직접 실행해 보기 (demo.py)

로봇(Jetson)에 센서를 연결하고 이 프로그램을 켜면, 화면에 계속 이렇게 찍힙니다:

```
거리: 512 mm   판정: safe
거리: 350 mm   판정: warn
거리: 150 mm   판정: stop
거리: (없음)    판정: stop   ← 센서 문제
```

손을 센서 앞에서 멀리/가까이 움직이며 단계가 잘 바뀌는지, 거리 기준이 적당한지
확인하고 `config.py` 값을 조절하면 됩니다. 끝낼 땐 `Ctrl-C` 를 누르면 깔끔하게 종료됩니다.

## 11. 문제가 생기면 (에러 처리)

- **센서 선을 못 열 때** (포트 권한/연결 문제): 프로그램이 그냥 죽지 않고,
  "센서 포트(`/dev/ttyTHS1`)를 열 수 없어요. 연결과 권한을 확인하세요" 같은
  **알아듣기 쉬운 메시지**를 보여 줍니다.
- **거리를 못 읽을 때**: 위에서 설명한 "모르면 멈춤" 규칙으로 알아서 처리됩니다.
- **프로그램 종료**: `Ctrl-C` 누르면 센서 연결을 정리하고 종료합니다.

## 12. 다른 프로그램이 이 결과를 쓰는 법 (참고)

이 모듈은 알려주기만 한다고 했죠. 나중에 로봇 운전 프로그램은 이렇게 씁니다
(예시일 뿐, 이 모듈에는 포함 안 됨):

```python
mon.tick()                       # 한 번 확인
if mon.verdict().level == "stop":
    전진_속도 = 0                # 앞으로 가는 명령을 0으로 (멈춤)
```
